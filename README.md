My best work usually starts with a messy system: scattered data, changing constraints, or a live operation that needs clearer decisions. I enjoy turning that controlled chaos into transparent workflows, useful models, and products people can actually operate.

## Selected projects

### [Football FC Research Simulator](https://github.com/jackholitza/football-fc-research-sim)

A TypeScript football-management research simulator that translates competition rules, roster constraints, financial regulations, and tactical decisions into an interactive product. Includes typed Web Worker simulation, Canvas rendering, IndexedDB persistence, and explicit verified/modeled/unknown data states.

**[Live demo](https://jackholitza.github.io/football-fc-research-sim/)**

### [World Cup 2026 Forecasting Model](https://github.com/jackholitza/WorldCupModel)

An interactive 48-team tournament model with player data, expected lineups, match forecasts, fair odds, Monte Carlo futures, and group/bracket scenario tools.

**[Live demo](https://jackholitza.github.io/WorldCupModel/)**

### [World Cup Officiating Monitor](https://github.com/jackholitza/world-cup-officiating-monitor)

A public monitoring dashboard that reconciles officiating, discipline, fixture, standings, and bracket data through deterministic match logic and Cloudflare Worker overlays.

**[Live demo](https://jackholitza.github.io/world-cup-officiating-monitor/)**
